## Quick Start

For new developers, start with getting started guide to set up your environment, review architecture documentation for system understanding, explore codebase structure, run tests to verify setup.

## Development Environment

Local development without Docker, hot reload for rapid iteration, test database and Redis, debug tooling enabled.

## Contribution Process

Fork repository, create feature branch, make changes following standards, write tests, submit pull request, code review and feedback.
