# Contributing Guide

Guide for contributing to the project.

## Contribution Process

### Before Starting
Check existing issues and pull requests, discuss significant changes first, ensure you understand the architecture, read code standards.

### Making Changes
Fork the repository, create feature branch, make focused changes, follow code standards, write tests for changes.

### Submitting Changes
Commit with clear messages, push to your fork, create pull request, describe changes clearly, link related issues.

### Review Process
Respond to review feedback, make requested changes, update documentation, ensure tests pass, wait for approval.

## Pull Request Guidelines

### PR Description
Describe what and why, reference related issues, list breaking changes if any, note testing performed.

### PR Size
Keep changes focused, one feature or fix per PR, split large changes, easier to review.

### Checklist
Tests added or updated, documentation updated, code follows standards, no linting errors, all tests pass.

## Code Review

Reviews ensure quality, provide feedback, catch issues early, share knowledge, maintain consistency.

## Issue Reporting

Clear description of issue, steps to reproduce, expected vs actual behavior, environment details, error messages if applicable.
